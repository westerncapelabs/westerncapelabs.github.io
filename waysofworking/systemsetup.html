System Setup

You have to run Linux or OSX when working at Western Cape Labs. If you run Linux, we highly recommend that you use a Debian-based distribution like Ubuntu or Linux Mint - you're on your own if you use something else.

Useful Software

Below are really useful things to install on a new system with some basic Linux installation instructions. As installation procedures change over time you might have to do something a bit differently. Generally the easiest way to install things is from the OS available packages, with:

sudo apt-get install thepackageyouwanttoinstall  // linux
brew install thepackageyouwanttoinstall  // mac

But you should probably read the latest install instructions online before doing any installation. If you do an installation and see any warning messages, stop and ASK!

Git

We use Git for software version control of all our projects. Github stores our code remotely. If you don't have a Github account, go make one now! If you have one already, make sure your email addresses are setup correctly - including adding your @westerncapelabs.com email address. Then do:

sudo apt-get install git
git config --global user.name yourgithubusername
git config --global user.email youremail

You can read up on setting up SSH Authentication, but to get started quickly you can just use HTTPS Authentication. However, to prevent having to enter your username and password every time, use a credential helper to set how often you get password-prompted to every 24 hours (86400 seconds)

git config --global credential.helper cache
git config --global credential.helper 'cache --timeout=86400'

Git Flow

Git flow is integral to the way we work. Try the following installation:

sudo apt-get install git-flow
// edit ~/.gitconfig to keep remote branches
git config --global gitflow.feature.finish.keepremote true

Hub

Hub allows for issue to pull-request conversion. Don't worry if you don't know what that means. Just try to install it

git clone https://github.com/github/hub.git
cd hub
script/build
mv ./hub /usr/local/bin  # move to somewhere in your PATH

Text Editor

Our two favourite editors are Sublime Text 3 and Atom. Sublime Text is loved by just about everyone who has used it, but it 'kind of' costs $70, while Atom is free and is practically a clone of Sublime Text. Sublime Text 'kind of' costs $70 because you can try it for free forever, you just have to ignore a pop-up 'buy me!' message every now and then. Or convince Mike to buy it for you :) If you haven't been using Sublime Text yet, Atom is probably your better bet.

All that being said, feel free to use another editor if you prefer. But you should install something good (Nano won't cut it!)

Python

Python is currently being used in two major versions: v2.x and v3.x. Mostly the differences don't really affect us, nowadays we tend to use v3.x on new projects.

Python may or may not be installed on your system, but that doesn't really matter, as you should be using pyenv! pyenv manages your Python versions, and with the extention pyenv-virtualenv, it also manages your virtual environments. More on virtual environments later. Install may happen like this:

git clone https://github.com/yyuu/pyenv.git ~/.pyenv
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init -)"' >> ~/.bashrc
exec $SHELL

git clone https://github.com/yyuu/pyenv-virtualenv.git ~/.pyenv/plugins/pyenv-virtualenv
echo 'eval "$(pyenv virtualenv-init -)"' >> ~/.bashrc
exec $SHELL

Pip

Pip is Python's package (library) installer/manager. You will use this all the time.

sudo apt-get install python-pip

Chrome

While you should feel free to use any browser, we mostly use Chrome. We use the Chrome app Postman a lot, not sure if that's available in Firefox. If not just install Chrome. Chrome may ask you to add a repository to your OS - it's okay to do so.
