Using Git Without git-flow'

Clone the code

$ git clone ...

Start a new feature branch based on develop (git flow feature start)

$ git checkout -b feature/issue-xx-abcd-efgh develop

Do the first push to origin(git flow feature publish)

$ git add ...
    $ git commit -m "..."
    $ git push origin feature/issue-xx-abcd-efgh

Push additional code

$ git add ...
    $ git commit -m "..."
    $ git push

Make pull request ...

Land a feature in develop (once pull request has received a +1)

(first make sure your local develop is up to date with origin)

$ git checkout develop
    $ git pull

(then merge your updated local develop into the feature branch)

$ git checkout feature/issue-xx-abcd-efgh
    $ git merge develop

(if there are any merge conflicts, resolve them, then push the merge)

$ git push

(now keep an eye on the website for travis being happy) (once travis is happy, merge in the feature branch)

$ git checkout develop
    $ git merge --no-ff feature/issue-xx-abcd-efgh
    $ git branch -d feature/issue-xx-abcd-efgh

(then push your local develop to origin)

$ git push
