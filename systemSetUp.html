Using Git Flow and Hub to handle issues

Task Commands
Install git flow on the terminal $ apt-get install git-flow
Make it work locally $ git flow init
Create a issue on github (e.g. Start blog application). $ git flow feature start issue-1-start-blog-application
Start coding and then:
  • $ git commit -am "all things"
  • $ git flow feature publish issue-1-start-blog-application
    		•
    Convert to a pull request $ hub pull-request -i 1 -b develop
    Send someone the pull-request link or @mention them
    Once its got a +1 from the reviewer(s) and tests are passing on Travis
  • $ git flow feature finish issue-1-start-blog-application
  • $ git push
    		•

    Git Commands

    Git Task Git Commands
    Tell Git who you are
  • git config --global user.name "Sam Smith"
  • git config --global user.email sam@example.com
    		•
    Create a new local repository git init
    Check out a repository
  • git clone /path/to/repository
  • git clone username@host:/path/to/repository
    		•
    Git add files
  • git add filename
  • git add * (* means select all)
    		•
    Commit git commit -am "Commit message"
    Push git push origin master
    Status git status
    Connect to a remote repository
  • git remote add origin server (configures a remote repository)
  • git remote -v (lists all remote repositries)
    		•
    Branches
  • git checkout -b branchname (configures a new branch and switch to it)
  • git checkout branchname (switches from one branch to another)
  • git branch(list all branches and tells you which branch you're working on)
  • git branch -d branchname (Deletes feature branch)
  • git push origin (push the branch to your remote repository to be used by others)
  • git push --all origin (push all branches to your remote repository)
  • git push origin :branchname (delete a branch in your remote repository)
